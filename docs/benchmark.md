# 벤치마크: 제타(Zeta) · 크랙(Crack) vs 우리 MVP (#8 Spike)

> 작성 2026-06-13 · 목적: 유명 AI 캐릭터 챗 서비스 대비 **빠진 기본기**를 식별하고 다음 sprint 백로그 후보를 산출한다. 수익화/구독/결제는 범위 제외.
> 방법: deep-research(웹 다중 검색 → 출처 fetch → 3표 적대 검증 → 합성). 일부 1차 출처(crack.wrtn.ai 402, namu.wiki 403)는 검색 캐시 의존 — 동적 화면은 시점차 가능. 신뢰도는 항목별 표기.

## 대상
- **제타(Zeta)** — 스캐터랩. 자체 LLM 기반. 대화→일러스트 변환이 특징. ([공식 블로그](https://blog.scatterlab.co.kr/zeta-intro-2506), [앱스토어](https://apps.apple.com/us/app/제타-zeta/id1619030760?l=ko), [탐색](https://zeta-ai.io/ko))
- **크랙(Crack)** — 뤼튼테크놀로지스. 2024-03 '캐릭터 챗'을 분리해 2025-04-03 정식 출시. 공식 고객센터 문서가 풍부. ([고객센터](https://help.crack.wrtn.ai/), [캐릭터 가이드](https://help.crack.wrtn.ai/crackguide/character))

## 1. 기능 비교

범례: ✅ 있음(1차 확인) · 🟡 부분/2차출처 · ❔ 미확인 · ❌ 없음

| 기능 | 제타 | 크랙 | 우리 MVP | 비고 |
|------|:---:|:---:|:---:|------|
| 메시지 스트리밍 | ✅ | ✅ | ✅ | 우리: SSE + 에러/타임아웃/safety-block 규약(#13) |
| 멀티턴 대화 | ✅ | ✅ | ✅ | 우리: 매 요청 full history 전송(stateless) |

| **답변 재생성** | ❔ | ✅ | 🟡 retry(에러시) | 우리는 에러 복구용 retry만, 정상 답변 재생성 없음 |

| 그룹채팅 | ❔ | ❔(파티챗 주장 반박됨) | ❌ | 양쪽 다 미확인 — 우선순위 낮음 |
| **멀티모달(이미지)** | ✅ 대화→일러스트 ~15초 | ❔ | ❌ | 제타 hallmark |
| 음성(TTS/STT) | ❔ | ❔ | ❌ | |
| 안전필터/콘텐츠 정책 | 🟡 성인 장르 태그 존재 | 🟡 안전/성인 작품 구분 | 🟡 prohibitions 필드만(미집행) | |

| 카테고리/태그 | ✅ 해시태그 ~50종(클릭·검색) | ✅ 장르 탭·키워드 | ❌ |
| 검색 | ✅ | ✅ | ❌ |

### 3) 페르소나 구조
| 항목 | 제타 | 크랙 | 우리 MVP |
|------|:---:|:---:|:---:|
| 생성 필드 | 이름·소개·성격·말투·세계관·예시대화·첫메시지·해시태그 | 5종 템플릿(기본/1:1 롤플/시뮬/생산성/커스텀) | 이름·tagline·성격·말투·세계관·예시대화·첫인사·**금지사항** |
| 사용자 제작 | ✅ 1분 제작, 400만+ | ✅ 코딩없이 텍스트 기반 | ✅ 템플릿→편집→저장 |
| **공개 공유** | ✅ 공개/비공개 선택 | ✅ 플랫폼 공유+크리에이터 커뮤니티 | ❌ localStorage 전용(비공개 고정) |
| 생성 템플릿 | 🟡 | ✅ 5종 | ✅ 5종 샘플 |

> 페르소나 **필드 구조 자체는 우리 MVP가 경쟁사와 거의 동등**(오히려 금지사항 필드는 우리 강점). 격차는 필드가 아니라 **공유·탐색·기억**에 있다.

## 2. Gap 분석 — 우리 MVP에 빠진 "기본기"

1. **장기기억/대화 요약** — 양사 공통 table-stakes. 우리는 full-history를 매번 보내 컨텍스트 한도·토큰비용에서 곧 한계. (제타조차 자체 LLM 50~60턴 + 요약주입으로 해결 중)
2. **캐릭터 서버 저장 + 공개 공유** — 우리 캐릭터는 localStorage 전용(비공개 고정). 양사는 공개 공유가 제품의 중심(탐색·랭킹·커뮤니티의 전제). 우리는 #14에서 **대화는 DB**에 넣었지만 **캐릭터는 아직 클라이언트**. 이게 풀려야 탐색/랭킹이 가능.
3. **캐릭터 탐색(검색·태그·랭킹·추천)** — 우리는 평면 목록. 양사는 풍부한 발견 경험. (2번이 선행)
4. **답변 재생성/편집/분기** — 크랙 1차 확인, UX 표준. 우리는 에러용 retry만.
5. **멀티모달(이미지)** — 제타의 차별화 포인트. 기본기보다는 차별화 영역, 비용·난이도 높음.
6. **안전필터 집행** — prohibitions 필드는 있으나 실제 정책/성인물 분리 없음. 공개 공유 도입 전 선결.

## 3. 다음 Sprint 백로그 후보 (우선순위)

| # | 항목 | 근거 | 규모(대략) | 선행 |
|---|------|------|:---:|------|
| B1 | **대화 요약/장기기억** (히스토리 요약 주입 + 토큰 가드) | 양사 공통 필수, full-history 한계 임박. #14 DB에 요약 컬럼 추가 자연스러움 | M | #14 ✅ |
| B2 | **캐릭터 DB 이전 + 공개/비공개** (localStorage→Postgres, browserId 소유) | 탐색·랭킹·공유의 전제. #14 인프라(Prisma/browserId) 재사용 | M | #14 ✅ |
| B3 | **캐릭터 탐색** (검색 + 태그/카테고리 + 인기 랭킹) | 양사 발견 경험의 핵심, 우리는 부재 | M~L | B2 |
| B4 | **답변 재생성 + 사용자 편집** | 크랙 표준 UX, retry 인프라 일부 재사용 | S~M | — |
| B5 | **안전필터/콘텐츠 정책** (prohibitions 집행 + 성인/일반 분리) | 공개 공유 전 선결 | M | B2 |
| B6 | **멀티모달 이미지**(대화→일러스트) | 제타 차별화. 기본기 아님 → 후순위 | L | — |

권장 다음 sprint 코어: **B1(기억) + B2(캐릭터 DB/공유) + B4(재생성·편집)**. B2가 B3/B5의 열쇠이므로 먼저 깐다.

## 4. 미해결 질문 (추가 조사 필요)
- 제타 그룹채팅(다중 캐릭터 동시) 존재 여부 — 미조사
- 양사 안전필터/콘텐츠 정책의 구체 동작(성인 분리, 미성년 보호, 금지어)
- 제타 음성 등 이미지 외 멀티모달 여부
- 제타의 정상 답변 재생성/편집/되감기 명시 제공 여부
- 크랙 생성 시 금지사항/네거티브 프롬프트 별도 필드 여부

## 5. 신뢰도 주의
- 크랙 핵심 출처(crack.wrtn.ai 402, namu.wiki 403) 일부 검색 캐시 의존 — 레일/랭킹은 동적, 시점차 가능
- 제타 50~60턴·요약주입은 2025-05 단일 블로그(vote 2-1), 현재(2026-06)까지 ~13개월 경과로 보완 레이어 발전 가능
- 제타 '15초 일러스트'는 벤더 마케팅 수치(독립 측정 아님)
- 그룹채팅 반박은 부정 증거 → '확실히 없음' 아닌 '확인 안 됨'
